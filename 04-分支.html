本地分支的操作 A, 本地分支的操作 -分支的概念 和开发中的应用 多人协作开发 主分支 功能分支（各个程序员 独立） 合并到主分支。 B,主分支 master 只有一个 用来保存和记录整个项目已完成的功能代码。 因此，不允许程序员直接在 master 分支上修改代码，因为这样做的风险太高，容易导致整个项目崩溃。 C,功能分支 由于程序员不能直接在 master 分支上进行功能的开发，所以就有了功能分支的概念。 功能分支指的是专门用来开发新功能的分支，它是临时从 master 主分支上分叉出来的，当新功能开发且测试 完毕后，最终需要合并到 master 主分支上 用完就删啦 D,查看分支列表 使用如下的命令，可以查看当前 Git 仓库中所有的分支列表： git branch branch 分支 注意：分支名字前面的 * 号表示当前所处的分支。 * master E,创建新分支 git branch 分支名字 基于当前分支创建一个新的分支 代码和当前分支一样，还是在当前分支 F,切换分支 git checkout 分支名字 G,快速创建并切换到分支 先切换到主分支 然后在主分支上创建一个分支 git checkout -b 分支名字 H,合并分支 先切换到要合并到的分支上，然后 git merge 加分支名 I 删除本地 分支 git branch -d 分支名 git branch -D reg 强制删除分支即使没有合并 合并分支之后 主分支上删分支 删除分支 不能在要删除的分支上 （基本操作是对文件进行本地仓库的添加提交） （分支是复制主分支的内容然后 进行多分支干活 修改提交到当前分支注意一定要在子分支提交修改代码 然后主分支合并。删除功能分支。 注意主分支不修改任何代码 ） J，遇到冲突时的分支的合并 （当前所在分支） git merge 在不同分支修改同一个文件 git 报错failed 需要打开冲突文件 vscode 提供了修改提示 点过之后 再次add . 提交即可 本地分支操作结束 二：远程分支操作。 1，将本地分支推送到远程仓库中的分支 在主分支上推送 分支（也不一定） 远程仓库默认只有一个主分支master -u是本地远程分支相关联 第一次 带-u 参数 之后直接 git push 因为远程仓库中有这个分支了直接push即可 在那个分支上吧。 ： 重命名 也可不加 不重命名 git push -u 远程仓库名字（origin） 本地分支名：远程分支名 2，查看远程仓库中的所有分支列表 git remote show 远程仓库的名字origin默认的 3. 下载远程分支 跟踪分支 ：把远程仓库中的分支下载到本地仓库中 git checkout 远程分支的名字 需要重命名时 git checkout -b 改的名字 远程仓库名/远程分支名 4，拉取远程分支的最新代码 拉取当前分支的最新代码 git pull 可在远程仓库中修改代码文件 点击对应文件 小铅笔修改 所以要拉取最新的 5，删除 远程分支 git push 远程仓库名 --delete 远程分支的名字 git push origin --delete 远程分支的名字
